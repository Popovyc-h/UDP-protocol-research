# UDP-protocol-research

### Завдання 1: Дослідження активних UDP з'єднань

**1. Не бачу ніяких станів для UDP сокетів, там немає "LISTENING" або "ESTABLISHED" тому що UDP не встановлює з'єднання взагалі. Тому колонка з станами пуста**

**2. У моїй системі UDP використовують такі порти: 
123 - W32Time, 
5050 - CDPSvc,
5353, 5355, 55472 - Dnscache,
1900, 54712, 54711 - SSDPSRV,
49664 - iphlpsvc**

**3. Браузер майже не використовує UDP для звичайного серфінгу тому що
HTTP/1, HTTP/2 вимагають надійної доставки — сторінка має завантажитись повністю і без помилок а з появою HTTP/3 браузер вже активно використовують UDP через QUIC**
